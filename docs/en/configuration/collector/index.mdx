# Configuring the Collector

The Alauda build of OpenTelemetry Operator utilizes a custom resource definition (CRD) to define the architecture and configuration settings for creating and deploying OpenTelemetry resources. You have the flexibility to deploy with default settings or customize the configuration to meet your specific requirements.

This section covers the essential aspects of configuring the OpenTelemetry Collector, including deployment strategies, component configuration, and resource management.
