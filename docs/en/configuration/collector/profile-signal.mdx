# Profile Signal

The OpenTelemetry Collector supports profiling signals, which enable continuous profiling data collection from your applications. Profiling provides deep insights into application performance by capturing detailed information about CPU usage, memory allocation, and execution patterns.

## Overview

Profile signals complement traditional observability data (traces, metrics, and logs) by offering granular visibility into application behavior at the code level. This capability helps identify performance bottlenecks, memory leaks, and inefficient code paths.

## Configuration

To enable profile signal collection in the OpenTelemetry Collector, you need to configure appropriate receivers that support profiling data formats. The Collector can ingest profiling data through various protocols and forward it to backend systems for analysis and visualization.

**Example configuration with profiling support:**

```yaml
apiVersion: opentelemetry.io/v1beta1
kind: OpenTelemetryCollector
metadata:
  name: profiling-collector
  namespace: observability
spec:
  mode: deployment
  config:
    receivers:
      otlp:
        protocols:
          grpc: {}
          http: {}
    exporters:
      otlp:
        endpoint: backend.observability.svc:4317
    service:
      pipelines:
        traces:
          receivers: [otlp]
          exporters: [otlp]
        metrics:
          receivers: [otlp]
          exporters: [otlp]
```

:::note
Profile signal support depends on the specific receivers and exporters you configure. Ensure your backend system supports profiling data ingestion and visualization.
:::

## Use Cases

Profile signals are particularly valuable for:

- **Performance optimization**: Identifying CPU-intensive functions and optimizing hot code paths
- **Memory analysis**: Detecting memory leaks and understanding allocation patterns
- **Latency investigation**: Correlating performance issues with specific code execution
- **Resource planning**: Understanding resource consumption patterns for capacity planning

By integrating profiling data with traces and metrics, you can achieve comprehensive observability across your application stack.
